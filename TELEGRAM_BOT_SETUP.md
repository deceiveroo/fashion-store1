## 1. Создание бота в Telegram

1. Найдите в Telegram бота [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания нового бота:
   - Придумайте имя бота (например, "Fashion Store Support Bot")
   - Придумайте юзернейм (должен заканчиваться на "bot", например: `fashion_store_support_bot`)
4. После создания BotFather предоставит вам **новый токен** (например: `123456789:AABBCCDDEEFFgghhIIJJKKLLMMNN`)
5. Сохраните этот токен, он понадобится для настройки

## 2. Настройка переменных окружения

Добавьте в ваш файл `.env.local` следующую строку:

```
TELEGRAM_BOT_TOKEN=ваш_новый_токен_который_дал_BotFather
```

**НИКОГДА не коммитьте токен в репозиторий и не публикуйте его в открытом виде**

## 3. Регистрация вебхука

После деплоя вашего приложения на сервер, выполните следующую команду в терминале (замените `your-actual-domain.com` на реальный домен):

```bash
curl -X POST "https://api.telegram.org/bot$TELEGRAM_BOT_TOKEN/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-actual-domain.com/api/telegram/webhook", "allowed_updates": ["message", "callback_query"]}'
```

Альтернативно, вы можете использовать онлайн-инструменты для отправки POST-запросов, заменив `$TELEGRAM_BOT_TOKEN` на реальный токен:

```
POST https://api.telegram.org/botYOUR_ACTUAL_TOKEN/setWebhook
Content-Type: application/json

{
  "url": "https://your-actual-domain.com/api/telegram/webhook",
  "allowed_updates": ["message", "callback_query"]
}
```

## 4. Структура файлов бота

- `telegram/bot.ts` - основная логика бота
- `app/api/telegram/webhook/route.ts` - маршрут для вебхука
- `app/api/health/route.ts` - эндпоинт для проверки состояния
- `lib/ai/support-processor.ts` - обработка запросов с помощью ИИ
- `components/chat/AIAssistantModal.tsx` - модальное окно ИИ-ассистента на сайте
- `components/chat/AIAssistantTrigger.tsx` - кнопка вызова ИИ-ассистента

## 5. Функциональность бота

- `/start` - начать взаимодействие с ботом
- `/help` - получить справку
- Режимы работы:
  - Режим ИИ (автоматические ответы)
  - Ожидание оператора (перевод запроса в очередь)
  - Режим человека (ответы от живого оператора)

## 6. Уведомления операторам

При получении новых запросов или при переводе статуса в "ожидание оператора", бот отправляет уведомления администраторам/операторам с кнопками:
- "Взять" - для принятия запроса
- "В очередь" - для оставления в очереди

## 7. Поддержка на стороне администратора

Администратор может управлять запросами через:
- `/admin/support` - панель управления запросами
- Возможность принимать запросы
- Отправлять ответы клиентам
- Возвращать запросы обратно к ИИ

## 8. Предотвращение "засыпания" бота

Для серверлесс сред (например, Vercel):
1. Используйте сервисы мониторинга (например, UptimeRobot) для периодического вызова `/api/health`
2. Это предотвратит "засыпание" функций и обеспечит быструю реакцию на сообщения
