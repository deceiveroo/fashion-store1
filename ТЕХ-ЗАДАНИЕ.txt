Цель: реализовать бесшовную систему поддержки, где бот обрабатывает запросы через LLM, а при низкой уверенности или по запросу пользователя диалог передаётся живому оператору через Telegram или веб-админку.
📋 1. Требования к функционалу
4.1. Уведомления
При новом диалоге или переходе в статус waiting_operator отправлять уведомление в личку админа/оператора или в Telegram-канал/топик.
Формат: 🔔 Новый запрос #ID | 👤 @username | 💬 "текст" | 🤖 уверенность: низкая | [👨‍💼 Взять] [📋 В очередь].
Кнопки — inline callback_query, обработка через вебхук.
Дедупликация: если диалог уже взят — не спамить.
4.2. Перехват диалога (Human-in-the-Loop)
Статусы диалога: ai → waiting_operator → human → closed.
Переключение возможно:
Из Telegram: оператор жмёт [Взять] в уведомлении → бот меняет статус и привязывает operator_id.
Из админ-панели: кнопка "Взять" в списке чатов.
Программно: при confidence < 2 или ключевых словах ("оператор", "человек", "помогите").
При статусе human все сообщения пользователя пересылаются оператору, ответы оператора (через reply в ТГ или форму в админке) идут пользователю.
4.3. Кнопка «Позвать оператора» в чате
Персистентная inline-кнопка в интерфейсе чата (или появляется после 2 неудачных ответов бота).
Логика: нажатие → статус waiting → бот: ⏳ Передаю специалисту... → уведомление операторам → кулдаун 10 мин для пользователя → авто-возврат в ai через 15 мин, если не взяли.
4.4. Качество ответов и fallback
Системный промпт: чёткая роль, ограничения, инструкция: «Если не уверен или вопрос требует доступа к аккаунту/оплате → отвечай шаблоном и предлагай оператора».
База знаний (RAG): документы → чанки → эмбеддинги (Supabase/pgvector или Qdrant) → поиск топ-3 → промпт с контекстом → LLM.
Оценка уверенности:
попросить LLM оценить 1-5 в JSON-блоке ответа,
эвристики (ключевые слова, длина, повтор),
низкий ранг поиска в RAG = низкая уверенность.
Fallback: при confidence <= 2 → статус waiting + шаблон: Не хочу вводить в заблуждение. Передаю вопрос специалисту.

import { pgTable, serial, text, timestamp, json, index, varchar, integer } from 'drizzle-orm/pg-core';
import { relations } from 'drizzle-orm';

export const chatSessions = pgTable('chat_sessions', {
  id: serial('id').primaryKey(),
  telegramUserId: integer('telegram_user_id').notNull(),
  username: text('username'),
  status: varchar('status', { length: 20 }).default('ai').notNull(), // ai | waiting | human | closed
  operatorId: integer('operator_id'),
  createdAt: timestamp('created_at').defaultNow().notNull(),
  updatedAt: timestamp('updated_at').defaultNow().notNull(),
}, (t) => [
  index('idx_telegram_user_id').on(t.telegramUserId),
  index('idx_status').on(t.status),
]);

export const messages = pgTable('messages', {
  id: serial('id').primaryKey(),
  sessionId: integer('session_id').references(() => chatSessions.id).notNull(),
  role: varchar('role', { length: 10 }).notNull(), // user | ai | operator
  content: text('content').notNull(),
  meta: json('meta'), // { confidence?: number, fallback?: boolean, messageId?: number, ragChunks?: string[] }
  createdAt: timestamp('created_at').defaultNow().notNull(),
});

export const operators = pgTable('operators', {
  id: serial('id').primaryKey(),
  telegramUserId: integer('telegram_user_id').unique().notNull(),
  name: text('name'),
  status: varchar('status', { length: 10 }).default('offline'), // online | busy | offline
  lastActiveAt: timestamp('last_active_at').defaultNow(),
});

export const sessionsRelations = relations(chatSessions, ({ many }) => ({
  messages: many(messages),
}));
export const messagesRelations = relations(messages, ({ one }) => ({
  session: one(chatSessions, { fields: [messages.sessionId], references: [chatSessions.id] }),
}));

bot.on('message:text', async (ctx) => {
  const text = ctx.message.text;
  const userId = ctx.from!.id;
  const username = ctx.from!.username;
  const session = await getOrCreateSession(userId, username);
  
  // Сохраняем сообщение пользователя
  await db.insert(messages).values({ sessionId: session.id, role: 'user', content: text });

  if (session.status === 'human') {
    // Пересылаем оператору
    const [op] = await db.select().from(operators).where(eq(operators.id, session.operatorId!));
    if (op) {
      await bot.api.sendMessage(op.telegramUserId, `💬 Клиент #${session.id} (@${username}):\n${text}`, {
        reply_parameters: { message_id: ctx.message.message_id }
      });
    }
  } 
  else if (session.status === 'ai') {
    // AI-обработка
    const aiResult = await processAIResponse(text, session.id);
    await ctx.reply(aiResult.text);
    await db.insert(messages).values({ 
      sessionId: session.id, role: 'ai', content: aiResult.text, 
      meta: { confidence: aiResult.confidence, fallback: aiResult.fallback } 
    });

    // Авто-fallback при низкой уверенности
    if (aiResult.fallback) {
      await updateSessionStatus(userId, 'waiting');
      await notifyOperators(userId, username || 'anon', ctx.chat.id, text);
    }
  }
  else if (session.status === 'waiting') {
    await ctx.reply('⏳ Ваш запрос уже в очереди. Ожидайте подключения оператора.');
  }
});

4. AI-процессор с fallback (lib/ai/support-processor.ts)
import { openai } from '@ai-sdk/openai';
import { generateText } from 'ai';
import { db } from '@/lib/db';
import { chatSessions } from '@/lib/db/schema/support';
import { eq } from 'drizzle-orm';

const SYSTEM_PROMPT = `Ты — помощник техподдержки интернет-магазина одежды "Fashion Store".
Правила:
1. Отвечай кратко, по делу, на русском языке.
2. Не выдумывай информацию: если нет данных в контексте — говори об этом.
3. Если вопрос касается: оплаты, возврата, личных данных, статуса заказа, доступа к аккаунту — сразу предлагай оператора.
4. В конце ответа добавь блок: \`\`\`json\n{"confidence": 1-5}\n\`\`\` где 1 = совсем не уверен, 5 = полностью уверен.
5. Если уверенность <= 2 — в тексте ответа напиши: "Передаю вопрос специалисту."`;

export async function processAIResponse(userMessage: string, sessionId: number) {
  // Опционально: загрузить контекст из истории (последние 5 сообщений)
  const history = await db.query.messages.findMany({
    where: eq(chatSessions.id, sessionId),
    orderBy: (m, { desc }) => [desc(m.createdAt)],
    limit: 5,
    columns: { role: true, content: true }
  }).then(rows => rows.reverse().map(r => `${r.role}: ${r.content}`).join('\n'));

  const result = await generateText({
    model: openai('gpt-4o-mini'),
    system: SYSTEM_PROMPT,
    prompt: `История диалога:\n${history || 'Нет истории'}\n\nПользователь: ${userMessage}`,
    temperature: 0.3,
    maxTokens: 500,
  });

  // Парсим confidence из JSON-блока
  const jsonMatch = result.text.match(/```json\n([\s\S]*?)\n```/);
  let confidence = 5;
  let cleanText = result.text;
  
  if (jsonMatch) {
    try {
      const meta = JSON.parse(jsonMatch[1]);
      confidence = typeof meta.confidence === 'number' ? Math.max(1, Math.min(5, meta.confidence)) : 5;
      cleanText = result.text.replace(jsonMatch[0], '').trim();
    } catch (e) {
      console.warn('Failed to parse confidence JSON', e);
    }
  }

  // Эвристики для понижения уверенности
  if (
    /(не знаю|не уверен|не могу|извините|не нашёл|нет информации)/i.test(cleanText) ||
    cleanText.length < 10 ||
    /(оплата|возврат|заказ|аккаунт|пароль|личный кабинет)/i.test(userMessage)
  ) {
    confidence = Math.min(confidence, 2);
  }

  return {
    text: cleanText || result.text,
    confidence,
    fallback: confidence <= 2
  };
}

  chats: ChatSession[];
  selectedChatId: number | null;

export const useSupportStore = create<SupportStore>((set) => ({
  chats: [],
  selectedChatId: null,
  setChats: (chats) => set({ chats }),
  setSelectedChat: (id) => set({ selectedChatId: id }),
  updateChat: (id, updates) => set((state) => ({
    chats: state.chats.map(c => c.id === id ? { ...c, ...updates } : c)

'use server';
import { db } from '@/lib/db';
import { chatSessions, messages, operators } from '@/lib/db/schema/support';
import { eq, and, desc, sql } from 'drizzle-orm';
import { revalidatePath } from 'next/cache';

export async function getActiveChats() {
  const rows = await db.select({
    id: chatSessions.id,
    telegramUserId: chatSessions.telegramUserId,
    username: chatSessions.username,
    status: chatSessions.status,
    operatorId: chatSessions.operatorId,
    updatedAt: chatSessions.updatedAt,
    lastMessage: messages.content,
  })
  .from(chatSessions)
  .leftJoin(messages, eq(messages.sessionId, chatSessions.id))
  .where(eq(chatSessions.status, 'waiting'))
  .orderBy(desc(chatSessions.updatedAt))
  .limit(50);
  
  return rows;
}

export async function takeChat(sessionId: number, operatorTelegramId: number) {
  // Находим или создаём оператора
  let [op] = await db.select().from(operators).where(eq(operators.telegramUserId, operatorTelegramId));
  if (!op) {
    [op] = await db.insert(operators).values({ telegramUserId: operatorTelegramId, status: 'online' }).returning();
  }
  
  await db.update(chatSessions)
    .set({ status: 'human', operatorId: op.id, updatedAt: new Date() })
    .where(eq(chatSessions.id, sessionId));
    
  revalidatePath('/admin/support');
}

export async function sendOperatorReply(sessionId: number, text: string, operatorTelegramId: number) {
  const [session] = await db.select().from(chatSessions).where(eq(chatSessions.id, sessionId));
  if (!session || session.status !== 'human') throw new Error('Invalid session');
  
  // Отправка через бота
  const botToken = process.env.TELEGRAM_BOT_TOKEN;
  if (botToken && session.telegramUserId) {
    await fetch(`https://api.telegram.org/bot${botToken}/sendMessage`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ chat_id: session.telegramUserId, text })
    });
  }
  
  // Сохранение в БД
  await db.insert(messages).values({ sessionId, role: 'operator', content: text });
  await db.update(chatSessions).set({ updatedAt: new Date() }).where(eq(chatSessions.id, sessionId));
}

export async function returnToBot(sessionId: number) {
  await db.update(chatSessions)
    .set({ status: 'ai', operatorId: null, updatedAt: new Date() })
    .where(eq(chatSessions.id, sessionId));
  revalidatePath('/admin/support');
}

7. Тестовый сценарий (E2E)
Пользователь пишет боту /start → видит кнопки.
Нажимает «Позвать оператора» → статус waiting, бот: ⏳ Передаю....
Оператор получает уведомление в ТГ с кнопкой [Взять].
