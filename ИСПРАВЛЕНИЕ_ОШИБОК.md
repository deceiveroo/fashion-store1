# 🔧 Исправление ошибки JWTSessionError

## Проблема
```
[auth][error] JWTSessionError
Failed query: select "role", "image" from "users" where "users"."id" = $1
params: a1929434-a0c0-4521-a4ab-051d129d4b6e,1
```

**Причина:** Пользователь с ID `a1929434-a0c0-4521-a4ab-051d129d4b6e` не существует в БД, но его сессия/токен еще активны.

---

## ⚡ Быстрое решение (3 минуты)

### Шаг 1: Выполните SQL скрипт очистки

Откройте Supabase SQL Editor:
🔗 https://supabase.com/dashboard/project/sjxepisvuthynvixpwii/sql

Скопируйте и выполните содержимое файла:
```
clean-and-reset.sql
```

Или скопируйте этот код:

```sql
-- Удаляем все сессии
TRUNCATE TABLE sessions CASCADE;

-- Удаляем старого администратора
DELETE FROM users WHERE email = 'admin@example.com';
DELETE FROM users WHERE id = 'a1929434-a0c0-4521-a4ab-051d129d4b6e';

-- Создаем нового администратора
INSERT INTO users (
  id, email, name, password, role, status, 
  email_verified, created_at, updated_at
) VALUES (
  gen_random_uuid(),
  'admin@example.com',
  'Администратор',
  '$2b$10$eDHm61JA35ergFkTomBxKOGLM0c663RjFr3GSbuOqqophMJnB714m',
  'admin',
  'active',
  NOW(),
  NOW(),
  NOW()
);

-- Проверка
SELECT id, email, role, status FROM users WHERE email = 'admin@example.com';
```

### Шаг 2: Очистите cookies в браузере

1. Откройте DevTools (F12)
2. Перейдите на вкладку **Application** (или **Приложение**)
3. В левом меню выберите **Cookies** → **http://localhost:3000**
4. Нажмите **Clear All** (или удалите все cookies вручную)

**Важно:** Это удалит старую сессию с несуществующим пользователем!

### Шаг 3: Перезапустите dev сервер

```bash
# Остановите сервер (Ctrl+C в терминале)
# Затем запустите заново:
npm run dev
```

### Шаг 4: Войдите заново

1. Откройте: http://localhost:3000/admin/login
2. Введите:
   - **Email:** `admin@example.com`
   - **Пароль:** `admin123`
3. Нажмите "Войти"

## 🔍 Дополнительная диагностика

### Если ошибка осталась:

#### 1. Проверьте что пользователь создан:

```sql
SELECT id, email, name, role, status, email_verified
FROM users 
WHERE email = 'admin@example.com';
```

**Ожидаемый результат:**
- `role` = `admin`
- `status` = `active`
- `email_verified` = текущая дата/время

#### 2. Проверьте что старый пользователь удален:

```sql
SELECT * FROM users WHERE id = 'a1929434-a0c0-4521-a4ab-051d129d4b6e';
```

**Ожидаемый результат:** Пусто (0 строк)

#### 3. Проверьте сессии:

```sql
SELECT * FROM sessions;
```

**Ожидаемый результат:** Пусто (0 строк) или только новые сессии

#### 4. Проверьте логи сервера:

После входа в логах должно быть:
```
[AUTH] authorize called
[AUTH] Email: admin@example.com
[AUTH] User found: true id=<новый-uuid>
[AUTH] Password match: true
[AUTH] Role: admin
[AUTH] Success! Returning user object
```

## 📊 Почему это произошло?

1. **Пользователь был удален из БД**, но его сессия осталась в cookies
2. **NextAuth пытается проверить сессию**, запрашивая данные пользователя
3. **Пользователь не найден**, NextAuth выдает `JWTSessionError`
4. **Запросы таймаутят**, потому что БД пытается найти несуществующего пользователя
