# Безопасность перед публикацией на GitHub

Перед тем как публиковать этот проект на GitHub, необходимо убедиться, что все чувствительные данные удалены или надежно защищены.

## Обнаруженные потенциальные проблемы безопасности

- В файле [app/api/seed/route.ts](file:///Users/NIKITA/Desktop/1/fashion-store/app/api/seed/route.ts) используются тестовые пароли `'admin123'` и `'user123'`

### 2. Рекомендации по исправлению

Прежде чем публиковать проект на GitHub:

1. Убедитесь, что все чувствительные данные (пароли, токены, ключи API) загружаются только из переменных окружения
2. Проверьте, что файлы `.env*` находятся в `.gitignore` и не будут отправлены в репозиторий
3. Обновите тестовые данные так, чтобы они использовали переменные окружения вместо жестко заданных значений

### 3. Проверка безопасности

Выполните следующие шаги перед публикацией:

- Проверьте все файлы на наличие жестко запрограммированных паролей, токенов или ключей API
- Убедитесь, что все чувствительные данные загружаются через `process.env.VARIABLE_NAME`
- Убедитесь, что в проекте присутствует файл `.env.example` с шаблонами для всех необходимых переменных
- Проверьте, что в `.gitignore` есть запись `.env*` для исключения всех файлов переменных окружения

## Безопасное развертывание

После публикации на GitHub:

1. Убедитесь, что в настройках репозитория отключена возможность публичного доступа к секретам
2. При развертывании на Vercel или других платформах настройте переменные окружения через панель управления
3. Регулярно меняйте пароли и ключи API в продакшен-окружении
