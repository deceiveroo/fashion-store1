## ✅ ШАГ 5: ОЧИСТИТЕ COOKIES (ОБЯЗАТЕЛЬНО!)

### В браузере:
1. Нажмите F12 (откроется DevTools)
2. Найдите вкладку "Application" (или "Приложение")
3. Слева найдите "Cookies"
4. Кликните на "http://localhost:3000"
5. Справа нажмите "Clear all site data" (или иконку корзины)

### Вы должны увидеть:
- Пустой список cookies
- Сообщение "No cookies"

**ВАЖНО:** Если не нашли "Application" - попробуйте вкладку "Storage"

## 🆘 ЕСЛИ НЕ РАБОТАЕТ

### Вариант 1: Используйте инкогнито

1. Откройте браузер в режиме инкогнито (Ctrl+Shift+N)
2. Перейдите: http://localhost:3000/admin/login
3. Войдите: admin@example.com / admin123

Если в инкогнито работает - проблема в cookies!

### Вариант 2: Проверьте SQL

Откройте Supabase SQL Editor и выполните:
```sql
SELECT * FROM users WHERE id = 'a1929434-a0c0-4521-a4ab-051d129d4b6e';
```

**Должно быть:** 0 rows (пусто)

Если пользователь еще есть - SQL не выполнился!

### Вариант 3: Проверьте нового администратора

```sql
SELECT id, email, role FROM users WHERE email = 'admin@example.com';
```

**Должно быть:** 1 row с НОВЫМ UUID (не a1929434...)
