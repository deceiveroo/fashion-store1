# Инструкции по обновлению репозитория на GitHub

## Подготовка к обновлению

Прежде чем обновлять репозиторий, убедитесь, что:

1. Все чувствительные данные находятся в файлах `.env*`, которые исключены из системы контроля версий (проверьте файл `.gitignore`)
2. В проекте есть файлы документации, такие как `GITHUB_README.md` и `SECURITY_CONSIDERATIONS.md`
3. Все зависимости правильно указаны в `package.json`
4. Все важные изменения закоммичены

## Шаги для обновления

### 1. Подготовка файлов

Убедитесь, что вы включили все нужные файлы:
- `GITHUB_README.md` - обновленный файл README для GitHub
- `SECURITY_CONSIDERATIONS.md` - информация о безопасности
- `package.json` и `package-lock.json` - зависимости проекта
- Все исходные файлы проекта
- Файлы конфигурации (`.gitignore`, `next.config.js`, `tsconfig.json` и др.)

### 2. Использование скриптов

Вы можете использовать один из предоставленных скриптов:

#### Linux/macOS:
```bash
./publish-to-github.sh
```

#### Windows:
```cmd
publish-to-github.bat
```

### 3. Ручное обновление

Если вы предпочитаете делать все вручную:

1. Убедитесь, что вы находитесь в корневой директории проекта
2. Установите удаленный репозиторий (если еще не сделали этого):
```bash
git remote add origin https://github.com/deceiveroo/fashion-store1.git
```

3. Убедитесь, что вы находитесь на правильной ветке:
```bash
git checkout main
```

4. Добавьте все изменения:
```bash
git add .
```

5. Сделайте коммит:
```bash
git commit -m "Update: Full project files for fashion store"
```

6. Загрузите изменения на GitHub:
```bash
git push -u origin main --force
```

## Важные замечания

- Убедитесь, что вы используете `--force` осторожно, так как это перезапишет содержимое репозитория
- Проверьте, что в `.gitignore` перечислены все чувствительные файлы
- Убедитесь, что в репозиторий не попадают временные файлы, файлы среды разработки и другие ненужные файлы
- Проверьте, что `GITHUB_README.md` содержит актуальную информацию о проекте
