# Исправление ошибок 401 (Unauthorized)

## Проблема
API роуты возвращают 401 ошибку:
- `/api/admin/stats` - 401
- `/api/admin/support-chats` - 401

## Причина
Функция `isAdmin()` в `lib/server-auth.ts` возвращала `boolean` вместо объекта пользователя.

## Решение

### 1. Обновлена функция `isAdmin()` в `lib/server-auth.ts`

**Было:**
```typescript
export async function isAdmin(): Promise<boolean> {
  const s = await getSession();
  return s?.user?.role === 'admin';
}
```

**Стало:**
```typescript
export async function isAdmin() {
  const s = await getSession();
  if (!s?.user || s.user.role !== 'admin') {
    return null;
  }
  return s.user;
}
```

Теперь функция возвращает:
- `null` - если пользователь не авторизован или не админ
- `user object` - если пользователь админ

### 2. Также обновлена функция `isStaff()`

```typescript
export async function isStaff() {
  const s = await getSession();
  if (!s?.user || !isStaffRole(s.user.role)) {
    return null;
  }
  return s.user;
}
```

## Проблема с AI

### Почему AI не работает?
В `.env` файле нет переменных для Cloudflare AI:
```env
CLOUDFLARE_ACCOUNT_ID=...
CLOUDFLARE_API_TOKEN=...
```

### Это нормально!
Система автоматически использует fallback режим с встроенными ответами.

### Как включить AI?

1. **Зарегистрируйтесь на Cloudflare**
   - https://dash.cloudflare.com/sign-up

2. **Включите Workers AI**
   - Перейдите в раздел "AI"
   - Включите Workers AI (бесплатно 10,000 запросов/день)

3. **Получите учетные данные**
   - Account ID: в URL после `/accounts/`
   - API Token: создайте в разделе "API Tokens"

4. **Добавьте в `.env`**
```env
CLOUDFLARE_ACCOUNT_ID=ваш_account_id
CLOUDFLARE_API_TOKEN=ваш_api_token
```

5. **Перезапустите сервер**
```bash
npm run dev
```

### Проверка работы AI

Откройте консоль сервера и отправьте сообщение в чат.

**С Cloudflare AI:**
```
[CHAT] Trying Cloudflare AI...
[CHAT] Cloudflare AI response: Привет! Чем могу помочь?
```

**Без Cloudflare AI (fallback):**
```
[CHAT] Cloudflare credentials not found
[CHAT] Using fallback response
```

## Fallback режим

Система отвечает на ключевые слова:
- "доставка" → информация о доставке
- "возврат" → информация о возврате
- "оплата" → информация об оплате
- "размер" → информация о размерах
- "заказ" → информация о заказе
- "скидка" → информация о скидках
- Другое → общая информация с контактами

## Если проблемы остались

### 401 на всех админ роутах
1. Проверьте, что вы вошли в систему
2. Проверьте роль пользователя в БД
3. Очистите cookies и войдите заново
4. Проверьте NEXTAUTH_SECRET в .env

### AI не отвечает
1. Это нормально без Cloudflare AI
2. Проверьте логи сервера
3. Должно быть "Using fallback response"
4. Fallback ответы должны работать

### Чат не открывается
1. Проверьте консоль браузера (F12)
2. Проверьте Network tab
3. Проверьте, что таблицы БД существуют
