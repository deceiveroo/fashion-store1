# Исправление чата поддержки - Полное руководство

## Что было исправлено

### 1. Файл админ панели (app/admin/support-chats/page.tsx)
**Проблема:** Файл был обрезан, отсутствовал весь UI
**Решение:** Создан полный компонент с:
- Списком всех чатов с фильтрацией (все/активные/решённые)
- Просмотром сообщений в реальном времени
- Возможностью отправки сообщений от админа
- Кнопками управления (взять под контроль, завершить, удалить)
- Автообновлением каждые 3 секунды

### 2. API роуты

#### app/api/admin/support-chats/route.ts
**Проблема:** Ошибка в DELETE методе - использовалась несуществующая переменная `session`
**Решение:** Удалена лишняя проверка, оставлена только проверка роли админа

#### app/api/admin/support-chats/[sessionId]/route.ts
**Проблема:** Неправильная сортировка сообщений (DESC вместо ASC)
**Решение:** Изменена сортировка на ASC для правильного отображения в чате

#### app/api/admin/support-chats/finish/route.ts
**Проблема:** Неправильный статус при завершении чата
**Решение:** Теперь ставит статус 'resolved' вместо 'active'

#### app/api/chat/route.ts
**Проблема:** Нет логирования для отладки AI
**Решение:** Добавлено подробное логирование всех этапов работы AI

### 3. Документация
Созданы файлы:
- `НАСТРОЙКА_AI_ЧАТА.md` - полное руководство по настройке AI
- `check-chat-tables.sql` - SQL скрипты для проверки БД

### Шаг 4: Тестирование пользовательского чата

1. Откройте сайт в браузере
2. Нажмите на кнопку чата (фиолетовый круг справа внизу)
3. Отправьте сообщение "привет"
4. Проверьте консоль браузера (F12) на ошибки
5. Проверьте логи сервера

**Ожидаемое поведение:**
- Сообщение отправляется
- Появляется индикатор загрузки (три прыгающие точки)
- Через 1-3 секунды приходит ответ от AI или fallback

**Если AI не работает:**
- Проверьте логи сервера
- Должно быть: `[CHAT] Cloudflare credentials not found` или `[CHAT] Using fallback response`
- Это нормально, если не настроен Cloudflare AI
- Система будет использовать встроенные ответы

**Ожидаемое поведение:**
- Список чатов загружается

## Устранение проблем

### Проблема: Чаты не загружаются в админке

**Решение:**
1. Проверьте авторизацию:
```sql
SELECT id, email, role FROM users WHERE role = 'admin';
```

2. Проверьте таблицы:
```sql
SELECT COUNT(*) FROM support_chat_sessions;
SELECT COUNT(*) FROM support_chat_messages;
```

3. Проверьте консоль браузера (F12) на ошибки

### Проблема: AI не отвечает (бесконечная загрузка)

**Причины и решения:**

1. **Нет Cloudflare AI**
   - Это нормально
   - Система должна использовать fallback
   - Проверьте логи: должно быть `[CHAT] Using fallback response`

2. **Ошибка в API**
   - Откройте консоль браузера (F12)
   - Перейдите на вкладку Network
   - Найдите запрос к `/api/chat`
   - Проверьте ответ

3. **Проблема с SSE (Server-Sent Events)**
   - Проверьте, что `/api/chat/stream` работает
   - В Network tab должен быть запрос типа `eventsource`
   - Если его нет - проблема с подключением

**Быстрое решение:**
```typescript
// В components/SupportChat.tsx можно временно отключить SSE
// и использовать прямой ответ из POST запроса

// Найдите эту часть:
const d = await r.json();
if (d.takenOver) {
  // ...
}
// Добавьте после неё:
if (d.message) {
  setMessages(prev => [...prev, {
    id: `ai-${Date.now()}`,
    text: d.message,
    sender: 'ai',
    timestamp: new Date(),
  }]);
  setLoading(false);
}
```

### Проблема: Сообщения не отправляются от админа

**Решение:**
1. Проверьте роль пользователя:
```sql
SELECT id, email, role FROM users WHERE email = 'ваш_email';
```

2. Если роль не 'admin', обновите:
```sql
UPDATE users SET role = 'admin' WHERE email = 'ваш_email';
```

3. Перелогиньтесь

### Проблема: Кнопка "Взять под контроль" не работает

**Решение:**
1. Проверьте API роут `/api/admin/support-chats/takeover`
2. Проверьте консоль браузера на ошибки
3. Проверьте, что поле `ai_disabled` существует в таблице:
```sql
SELECT column_name FROM information_schema.columns 
WHERE table_name = 'support_chat_sessions' AND column_name = 'ai_disabled';
```

## Настройка Cloudflare AI (опционально)

### Зачем нужно?
- Умные ответы на вопросы пользователей
- Понимание контекста
- Более естественный диалог

### Как настроить?

1. **Зарегистрируйтесь на Cloudflare**
   - https://dash.cloudflare.com/sign-up

2. **Включите Workers AI**
   - Перейдите в раздел "AI"
   - Включите Workers AI (бесплатно 10,000 запросов/день)

3. **Получите учетные данные**
   - Account ID: в URL после `/accounts/`
   - API Token: создайте в разделе "API Tokens"
     - Шаблон: "Edit Cloudflare Workers"
     - Права: Workers AI:Edit

4. **Добавьте в .env**
```env
CLOUDFLARE_ACCOUNT_ID=ваш_account_id
CLOUDFLARE_API_TOKEN=ваш_api_token
```

5. **Перезапустите сервер**
```bash
npm run dev
```

6. **Проверьте логи**
Должно быть:
```
[CHAT] Trying Cloudflare AI...
[CHAT] Cloudflare AI response: ...
```

## Настройка Telegram уведомлений (опционально)

### Зачем нужно?
- Админ получает уведомления о новых чатах
- Можно отвечать прямо из Telegram

### Как настроить?

1. **Создайте бота**
   - Напишите @BotFather в Telegram
   - Команда: `/newbot`
   - Получите токен

2. **Получите Chat ID**
   - Напишите боту любое сообщение
   - Откройте: `https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates`
   - Найдите `"chat":{"id":123456789}`

3. **Добавьте в .env**
```env
TELEGRAM_BOT_TOKEN=ваш_bot_token
TELEGRAM_CHAT_ID=ваш_chat_id
```

4. **Перезапустите сервер**

## Мониторинг

### Проверка активности чатов
```sql
-- Активные чаты за последние 24 часа
SELECT 
    session_id,
    user_name,
    message_count,
    last_message_at
FROM support_chat_sessions
WHERE last_message_at > NOW() - INTERVAL '24 hours'
ORDER BY last_message_at DESC;
```

### Статистика по AI
```sql
-- Сколько сообщений обработал AI vs админ
SELECT 
    sender,
    COUNT(*) as count,
    COUNT(DISTINCT session_id) as unique_sessions
FROM support_chat_messages
WHERE created_at > NOW() - INTERVAL '7 days'
GROUP BY sender;
```

### Проверка производительности
```sql
-- Среднее время ответа (если есть timestamps)
SELECT 
    AVG(EXTRACT(EPOCH FROM (
        SELECT MIN(created_at) 
        FROM support_chat_messages scm2 
        WHERE scm2.session_id = scm1.session_id 
        AND scm2.sender IN ('ai', 'admin')
        AND scm2.created_at > scm1.created_at
    ) - scm1.created_at)) as avg_response_time_seconds
FROM support_chat_messages scm1
WHERE sender = 'user'
AND created_at > NOW() - INTERVAL '7 days';
```
