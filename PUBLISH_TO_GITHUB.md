# Инструкции по публикации проекта на GitHub

## Шаг 1: Создание нового репозитория на GitHub

1. Перейдите на https://github.com/new
2. Введите имя репозитория (например, `fashion-store`)
3. Выберите тип "Public" или "Private" по вашему усмотрению
4. НЕ ставьте галочку "Initialize this repository with a README"
5. НЕ добавляйте .gitignore или license (они уже есть в вашем проекте)
6. Нажмите "Create repository"

## Шаг 2: Инициализация локального репозитория и публикация

Откройте терминал или командную строку в директории проекта (`c:\Users\NIKITA\Desktop\1\fashion-store`) и выполните следующие команды:

```bash
# Инициализация Git-репозитория
git init

# Добавление всех файлов к коммиту
git add .

# Создание первого коммита
git commit -m "Initial commit: Fashion Store project"
```

## Шаг 3: Связывание с удалённым репозиторием и публикация

Замените URL ниже на URL вашего созданного репозитория на GitHub:

```bash
# Установка удаленного репозитория
git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИТОРИЯ.git

# Переименование локальной ветки в main (по умолчанию GitHub)
git branch -M main

# Отправка проекта на GitHub
git push -u origin main
```

## Важная информация

- Файлы `.env`, `.env.local` и другие файлы с чувствительными данными автоматически игнорируются Git благодаря правилам в файле `.gitignore`.
- Файл `.env.example` будет опубликован, чтобы другие разработчики могли видеть структуру необходимых переменных окружения.
- Убедитесь, что вы не добавляете реальные секреты в репозиторий.

## Если возникнут ошибки

Если Git ругается на конфликты или другие проблемы:

1. Убедитесь, что вы не инициализировали репозиторий с README на GitHub
2. Если у вас уже есть какие-то коммиты на GitHub, возможно, потребуется выполнить:
   ```bash
   git pull origin main --allow-unrelated-histories
   ```
   Затем снова:
   ```bash
   git push -u origin main
   ```

Проект успешно опубликуется на GitHub с соблюдением всех мер безопасности.
